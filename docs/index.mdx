# n8n Community Node for Crossmint

This community node for n8n provides a complete integration with Crossmint's **Wallet**, **Checkout**, and **NFT** APIs. It lets users and AI agents program digital money inside wallets and automate the purchase of physical products‚Äîall inside your n8n workflows.

## üöÄ Installation (Local Development Setup)

For now you have to run the Crossmint node from source:

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/Crossmint/n8n-nodes-crossmint.git
    cd n8n-nodes-crossmint
    ```
  </Step>

  <Step title="Install dependencies and build">
    ```bash
    npm install
    npm run build
    ```
  </Step>

  <Step title="(Optional) Run in development with auto-reload">
    ```bash
    sudo npm run dev
    ```
  </Step>
</Steps>

<Callout type="info">

</Callout>

## üí° Supported Operations

The node is organized into three primary resources: **Wallet**, **Checkout**, and **NFT**.

### Resource: Wallet

Operations for managing blockchain wallets that can hold and transfer money (e.g., USDC).

- **Get or Create Wallet** ‚Äî Creates a non-custodial smart wallet or retrieves an existing one. You control it via a private key that authorizes all transactions. Idempotent.
  - **Chain Type** ‚Äî Blockchain type (`evm` for Ethereum-compatible, `solana` for Solana)
  - **Owner Type** ‚Äî Optional owner identifier type (`email`, `userId`, `phone`, `twitter`, `x`, `none`)
  - **Owner Details** ‚Äî Specific owner information based on the selected type
  - **Admin Signer** ‚Äî Private key that authorizes all transactions from this wallet (see **Admin Signer Private Key** below)

- **Get Wallet** ‚Äî Retrieves wallet details using its locator (address, email, user ID, etc.).
  - **Wallet** ‚Äî Wallet identifier (`address`, `email`, `userId`, `phone`, `twitter`, `x`)
  - **Chain Type** ‚Äî Required for non-address locators (`evm` or `solana`)

- **Create Transfer** ‚Äî Sends tokens (like USDC) or NFTs between wallets. Requires the wallet‚Äôs private key to sign and authorize.
  - **Blockchain Type** ‚Äî `evm` or `solana`
  - **Origin Wallet** ‚Äî Source wallet
  - **Recipient Wallet** ‚Äî Destination wallet
  - **Token Chain** ‚Äî Specific network (e.g., `ethereum-sepolia`)
  - **Token Name** ‚Äî Token symbol/identifier (e.g., `usdc`)
  - **Transfer NFT** ‚Äî Toggle for NFT transfers (hides amount)
  - **Amount** ‚Äî Token amount (hidden for NFTs)

- **Get Balance** ‚Äî Checks token balances for a wallet across one or more networks.
  - **Wallet** ‚Äî Locator (`address`, `email`, `userId`, `phone`, `twitter`, `x`)
  - **Chain Type** ‚Äî Required for non-address locators (`evm` or `solana`)
  - **Chains** ‚Äî e.g., `ethereum-sepolia`, `polygon`, `base`
  - **Tokens** ‚Äî Comma-separated list of token symbols

- **Sign Transaction** ‚Äî Signs a transaction with a private key and submits the signature to complete pending transactions.
  - **Chain** ‚Äî Network for signing
  - **Wallet Address** ‚Äî From the Create Transfer response
  - **Transaction ID** ‚Äî ID that needs approval
  - **Transaction Data** ‚Äî Hash/message to sign from transfer response
  - **Signer Address** ‚Äî Address of the external signer
  - **Signer Private Key** ‚Äî Private key to sign the transaction
  - **Wait for Completion** ‚Äî Poll until final status

### Resource: Checkout

Operations to automate purchasing products using digital money (e.g., USDC). Two-step flow:

- **Create Order**
  - **Platform** ‚Äî `amazon` or `shopify`
  - **Product Identifier** ‚Äî Product URL or ASIN/ID (e.g., Amazon URL or `B01DFSADS2`)
  - **Recipient Email** ‚Äî For order confirmation
  - **Recipient Name** ‚Äî Full name for shipping
  - **Address Line 1/2**, **City**, **State** (optional), **Postal Code**, **Country**
  - **Environment** ‚Äî Staging/Testnet or Production/Mainnet
  - **Payment Method** ‚Äî Blockchain network
  - **Payment Currency** ‚Äî e.g., `usdc`
  - **Payer Address** ‚Äî Wallet address that will pay

- **Pay Order**
  - **Serialized Transaction** ‚Äî From Create Order response
  - **Payment Chain** ‚Äî Network for payment
  - **Payer Wallet Address** ‚Äî Paying wallet
  - **Private Key** ‚Äî Authorize payment

### Resource: NFT

- **Mint NFT** ‚Äî Create a new NFT and send it to a wallet. Supports templates, metadata URLs, or object metadata.
  - **Collection ID**
  - **Recipient** ‚Äî (`address`, `email`, `userId`, `phone`, `twitter`, `x`)
  - **Chain**
  - **Metadata Type** ‚Äî `template`, `url`, or `object`

  **When Metadata Type = "Metadata Object":**
  - **NFT Name** (max 32)
  - **NFT Image URL**
  - **NFT Description** (max 64)
  - **Animation URL** (EVM only, optional)
  - **Symbol (Solana)** (max 10, optional)
  - **Attributes** (JSON array, optional)

  **When Metadata Type = "Metadata URL":**
  - **Metadata URL** ‚Äî JSON file URL

  **When Metadata Type = "Template ID":**
  - **Template ID**

  **Common fields:**
  - **Send Notification**, **Locale**, **Reupload Linked Files**
  - **Compressed (Solana)** ‚Äî Use compressed NFTs (Solana only)

- **Get NFTs From Wallet** ‚Äî Retrieves all NFTs owned by a wallet, with optional contract/token filters.
  - **Wallet Identifier**
  - **Chain**
  - **Contract Addresses** ‚Äî Comma-separated (optional)
  - **Token ID** ‚Äî Specific token (optional)

## üîë Understanding Wallet Locators

Wallet locators are used throughout Crossmint operations. They provide a flexible way to identify wallets with different identifiers.

### Locator Types

| Type | Format | Example | Use Case |
|------|--------|---------|----------|
| **Wallet Address** | `0x...` | `0x1234567890123456789012345678901234567890` | Direct blockchain address reference |
| **Email** | `email:{email}:{chainType}:smart` | `email:user@example.com:evm:smart` | User identification by email |
| **User ID** | `userId:{id}:{chainType}:smart` | `userId:user-123:evm:smart` | Custom user identifier |
| **Phone Number** | `phoneNumber:{phone}:{chainType}:smart` | `phoneNumber:+1234567890:evm:smart` | SMS-based identification |
| **Twitter Handle** | `twitter:{handle}:{chainType}:smart` | `twitter:username:evm:smart` | Social media identification |
| **X Handle** | `x:{handle}:{chainType}:smart` | `x:username:evm:smart` | X (formerly Twitter) identification |

For more details, see <a href="https://docs.crossmint.com/api-reference/wallets/get-wallet-by-locator">Wallet Locators</a>.

### Chain Types

- **EVM** ‚Äî Ethereum-compatible chains (Ethereum, Polygon, Base, Arbitrum, etc.)
- **Solana** ‚Äî Solana blockchain

### Best Practices

1. **Email locators** are great for user-friendly identification
2. **Wallet addresses** provide direct blockchain access
3. **User ID locators** fit existing user systems
4. Always specify the correct **chain type** for non-address locators

## üîê Admin Signer Private Key

The **Admin Signer Private Key** is a critical security component that gives you full control over your Crossmint wallet.

### What is an Admin Signer?

The Admin Signer is the private key that acts as the ‚Äúmaster key‚Äù for your smart wallet. It‚Äôs used to:
- Authorize all transactions
- Sign transfer approvals
- Control wallet permissions and operations

### Key Requirements by Blockchain

**EVM (Ethereum, Polygon, Base, etc.)**
- 32-byte hexadecimal string
- Example: `0x1234567890123456789012345678901234567890123456789012345678901234`
- With or without `0x` prefix

**Solana**
- Base58 encoded string
- Typically 64 bytes when decoded
- Example: `5Kb8kLf9CJtPkDCe4jfE9TjC8d7X9e3Jh4F6h8F2K3h7J9F4K6h8F2K3h7J9F4K6h`

### Generating Private Keys

**Online Tools**
- **Crossmint Generator**: <a href="https://crypto-address-generator.val.run/">https://crypto-address-generator.val.run/</a>
- **MetaMask**: Export a private key from an existing wallet

**Local/Terminal (recommended for production)**

_EVM_
```bash
# Node.js
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

# OpenSSL
openssl rand -hex 32

# Python
python3 -c "import secrets; print(secrets.token_hex(32))"
```

_Solana_
1. Install Solana CLI ‚Äî <a href="https://solana.com/docs/intro/installation">Installation guide</a>
2. Generate a keypair:
```bash
solana-keygen new --outfile ~/my-wallet.json --no-bip39-passphrase
cat ~/my-wallet.json
```

**Hardware Wallets**
- Export/derive keys securely from Ledger, Trezor, etc.

<Callout type="warning">
<b>Security best practices</b>
<ol>
  <li>Never share your private key</li>
  <li>Store in environment variables or secure vaults</li>
  <li>Use different keys for staging vs production</li>
  <li>Back up securely (multiple locations)</li>
  <li>Test with small amounts on testnet first</li>
  <li>Rotate keys periodically</li>
</ol>
</Callout>

### How It Works in Crossmint

1. **Wallet Creation** ‚Äî Your private key becomes the admin signer for the smart wallet
2. **Transaction Flow** ‚Äî Create transaction ‚Üí Crossmint generates approval ‚Üí Sign with private key ‚Üí Submit signature
3. **Non-Custodial** ‚Äî You control the key and the wallet
